# 今日 · 桌面日程记事本

一个零依赖、数据保存在本机的日程与工作记录原型。

## 使用方式

安装完成后，双击桌面的“今日日程”快捷方式即可启动。应用采用透明无边框窗口：

- “玻璃模式”使用单层透明窗口，桌面壁纸贯穿表头、任务栏和日程区
- “低干扰”仅降低界面不透明度，不会屏蔽鼠标，仍可立即编辑待办
- 按 `Ctrl + Shift + Space`、双击系统托盘图标或使用托盘菜单可恢复正常显示

也可以双击 `启动今日日程.cmd` 启动，或直接打开 `index.html` 使用网页版。

## 已实现

- 默认显示本周周一至周五，可前后切换周或直接选日期
- 每个日期单独保存待办、日程和当天备注
- 待办可勾选完成、关闭、恢复
- 待办包含目标完成日期与时间、责任人、任务说明
- 待办包含优先级、计划中/进行中状态与完成进度
- 优先级支持 KPI、跟踪关注、重要紧急、中止暂停，并可随时编辑
- 优先级增加“一般日常”，并作为新任务默认值
- 可勾选“每月固定时间”，保存为月度规则；每个自然月打开/进入该月时只生成当月任务实例
- 遇到没有对应日期的短月份时，自动使用该月最后一天
- 月度子任务会自动归属同月份的月度主计划实例
- 桌面窗口支持自由缩放，窄窗口会自动压缩布局
- 没有关联日程的任务严格归为“计划中”，拖入日程后才归为“进行中”
- 排入未来日程的任务仍保持“计划中”并留在左侧；到达开始时间后才自动变为“进行中”
- 实际开始时间取最早关联日程的开始时刻
- 投入工时只计算已经经过的时段：未来为 0，进行中按当前时刻截算，完成后按完成时间截算
- 已完成与已关闭在待办筛选中统一归为“已结束”
- 待办首次拖入日程后自动开始计时，并记录真实开始时间
- 父任务必须填写业务背景与问题原因，所有任务包含交付进度说明
- 新建任务时交付进度说明为选填；任务进行后可通过表头“更新进度”集中回顾填写
- 更新进度工作台可批量调整进行中任务的百分比与交付说明，达到 100% 自动完成
- 自动记录任务创建时间、最后更新时间、实际完成时间及累计投入工时
- 待办可设为主计划，或归属某个主计划成为子计划
- 待办支持日、周、月三种查看方式
- 待办与日程均可重新编辑和删除
- 可选择导出 Excel 或 JSON；Excel 包含任务清单、日程记录、每日备注三个工作表
- 导出字段已为后续甘特图展示预留
- 表头统一切换日、周、月视图，并同步切换左侧任务范围和右侧日历
- 将进行中的待办拖到右侧小时段，自动生成 1 小时日程
- 点击任一小时段添加日程，支持半小时粒度、编辑、删除和颜色分类
- 自动统计当天计划时长、已记录时长、空闲时间与任务完成率
- 当前日期显示实时红线
- 紧凑/舒展两种时间轴密度
- 所有数据通过浏览器 `localStorage` 与桌面应用独立数据文件双重保存在本机

## 数据保留与升级约定

任务、日程和备注属于用户数据，后续发布功能更新时不要删除或重置。

当前应用会同时保存两份数据：

- 浏览器本地存储：用于页面即时读写。
- 桌面应用独立数据文件：`%APPDATA%\today-daily-planner\planner-data.json`，用于跨版本保留记录。

每次覆盖独立数据文件前，应用会在 `%APPDATA%\today-daily-planner\backups` 自动保留备份：

- `planner-data-latest.json`：最近一次完整数据。
- `planner-data-before-update-*.json`：覆盖前的数据快照。

以后修改程序时，只更新 `index.html`、`styles.css`、`app.js`、`main.js`、`preload.js` 等程序文件；不要清空 localStorage，不要更换 `STORAGE_KEY`，不要删除 `%APPDATA%\today-daily-planner` 下的数据文件。
